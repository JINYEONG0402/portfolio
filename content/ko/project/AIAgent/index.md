title: AI 강사 Agent 구축
date: 2025-11-12

ppt자료를 넣으면 AI가 설명해주는 영상으로 만들어줌 !


<!--more-->

저는 PPT만 업로드하면 자동으로 강의 영상을 제작하는 AI 강사 Agent를 설계·구현했습니다. 
이 시스템은 슬라이드 분석, 음성 생성(TTS), 영상 렌더링, 최종 강의 영상 병합까지의 전 과정을 자동화하는 것을 목표로 합니다. 
먼저 PPT 파일로부터 텍스트, 이미지, 표 등 핵심 정보를 추출하고 이를 기반으로 슬라이드별 강의 스크립트를 생성합니다. 
이후 OpenAI TTS 모델을 활용해 슬라이드마다 자연스러운 음성을 생성하고, 
ffmpeg를 이용해 해당 슬라이드 이미지와 음성을 결합해 개별 강의 영상을 만듭니다. 
모든 슬라이드의 영상이 생성되면 이를 순차적으로 병합하여 하나의 완성된 강의 영상을 자동으로 생성합니다.

전체 프로세스는 LangGraph 기반 노드 구조로 설계되어, TTS 생성 → 슬라이드 영상 생성 → 인덱스 증가 → 반복 처리 → 최종 영상 합치기로 
이어지는 흐름을 안정적으로 제어합니다. 
이러한 구조 덕분에 강의 길이와 슬라이드 수에 관계없이 확장성이 뛰어나며, 에러 발생 시에도 개별 노드에서 상황을 처리해 시스템 
안정성을 높였습니다.

이 프로젝트를 통해 반복적이고 시간이 많이 소요되는 강의 제작 과정을 자동화함으로써, 강의자나 교육 기관이 더 빠르고 
효율적으로 콘텐츠를 생산할 수 있는 기반을 마련했습니다. AI 기술과 미디어 처리 기술을 실제 서비스 형태로 구현한 경험이라는 점에서 
의미 있었다고 생각합니다.


저는 전체 시스템에서 슬라이드 단위로 생성된 이미지와 오디오를 정확하게 합쳐 영상으로 만드는 핵심 모듈을 담당했습니다. 
이 과정은 단순히 ffmpeg로 두 파일을 묶는 수준이 아니라, 슬라이드 인덱스 관리, 파일 경로 생성, 
음성 길이에 맞춘 영상 렌더링 등 세부적인 요소들을 정교하게 제어해야 했습니다.
특히 구현 과정에서 여러 기술적 난관이 있었습니다. 슬라이드 인덱스가 반복 중에 꼬이거나 배열 범위를 벗어나며 IndexError가 발생하는 문제, TTS 단계에서 생성된 오디오 파일 경로가 정상적으로 state에 저장되지 않아 node_make_video에서 음성을 찾지 못하는 문제, 슬라이드 이미지를 PNG로 추출하는 과정에서 예상치 못하게 placeholder 이미지가 사용되거나 경로가 깨지는 문제 등이 반복적으로 발생했습니다. 또한 ffmpeg 실행 시 잘못된 입력 경로, 파일 생성 순서 오류, 음성 길이와 영상 길이가 일치하지 않아 영상이 갑자기 끊기거나 
오류를 내는 이슈도 해결해야 했습니다.
LangGraph 기반으로 노드가 분리되어 있다 보니, 각 노드가 주고받는 state의 구조가 조금만 틀어져도 
전체 파이프라인이 실패하는 경우가 많았습니다. 그래서 오디오 파일 리스트 구조, 슬라이드 배열 구조, 
작업 디렉토리 생성 로직 등을 하나씩 점검하며 시스템 안정성을 높여가는 데 많은 시간이 필요했습니다. 
이러한 디버깅 과정이 가장 도전적인 부분이었지만, 동시에 프로젝트 구조를 깊이 이해하게 되는 계기도 되었습니다.

결과적으로 슬라이드별 이미지·오디오가 안정적으로 결합되어 일관된 강의 영상을 생성할 수 있도록 
모듈을 완성하였고, 최종적으로 전체 강의 영상 병합까지 자연스럽게 이어지는 부드러운 제작 파이프라인을 구축하는 데 기여했습니다.
